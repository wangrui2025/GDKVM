# 特别的

## 内容
1. 将 shell 代码块中的注释移到了代码块上方作为说明文本，同时保持了 pyproject.toml 的内容不变。所有的 Shell 命令现在都标记为 bash，配置文件标记为 toml，以实现语法高亮。
2. 在高质量的技术文档（尤其是 SOP 标准作业程序）中，确实应该追求**“零噪音”**。那句话带有主观评价色彩（“虽然...但是...”、“合理的”），不仅显得啰嗦，还隐含了一种“邀功”或“说教”的意味，对于读者来说确实属于干扰信息。真正的工程师文档应当是冷峻、客观、陈述事实的。
3. 把出现的“/wr2024”这种匿名化“/Anon”

## 美工
1. 删除了标题中文本自带的 Emoji 图标（⚠️、🔍、🔧），避免与前面的 FontAwesome 样式图标重复。
2. 文本段落中的行内代码（例如 bash、yaml、pyproject.toml 等）将会有高亮背景和独特的文字颜色。
3. 配置代码块：为各个 `<pre><code>` 块添加了具体的语言标识（如 language-bash、language-toml），现在代码块会显示漂亮的语法颜色。
4. 完成一项，标题前面可以点击，变为绿色对勾。
